## 에러 기록

### 2026-07-17 [Playwright Browser Missing Error / 환경 에러]
- **발생 위치**: `server.py:256` 및 `pytest` 수행 시
- **증상**: Playwright 브라우저 기동(`launch`) 시점에 `Executable doesn't exist` 오류가 발생하며 브라우저 기동 실패 및 관련 단위 테스트 전체 실패.
- **원인**: 가상환경이 동기화/재구축되면서 플레이라이트 크로미움 헤드리스 브라우저 바이너리가 로컬 가상환경 내부 캐시 경로에 정상적으로 내려받아지지 않은 상태였음.
- **해결**: `uv run playwright install` 명령을 기동하여 필요한 브라우저 실행 바이너리를 다운로드 및 구축함.
- **상태**: 해결

### 2026-07-17 [MagicMock Await TypeError / 단위 테스트 에러]
- **발생 위치**: `test_integration_v2.py` 내 Mock 서버 세션 테스트 수행 시 `server.py` 호출부
- **증상**: `TypeError: object MagicMock can't be used in 'await' expression` 발생.
- **원인**: 단위 테스트에서는 `server._ensure_session` 및 `server._try_init_session`을 patch(Mock)하여 제어하고 있었으나, 멀티 계정 기능이 도입되면서 서버 코드가 `_ensure_session_for` 및 `_try_init_session_for`를 직접 기동하면서 Mocking을 우회하여 실제 비즈니스 로직(Mock context 상의 await)이 기동됨.
- **해결**: `account_id` 파라미터가 비어있는 기본 계정 모드에서는 기존 Mocking 대상인 `_ensure_session()`과 `_try_init_session()`을 경유하여 동작하도록 리팩토링함.
- **상태**: 해결

### 2026-07-17 [Mock get_page Argument Match TypeError / 단위 테스트 에러]
- **발생 위치**: `test_integration_v2.py::TestServerRouting` 테스트 기동 시
- **증상**: `TypeError: fake_get_page() takes 0 positional arguments but 1 was given`으로 인한 테스트 크래시.
- **원인**: `NaverBlogMCPServer.get_page(self, account_id: Optional[str] = None)` 메서드에 멀티 계정 대응 인수(`account_id`)를 받도록 규격이 추가되면서, 테스트 코드에서 0개의 인자값만 받도록 설계된 `async def fake_get_page()` Mock 함수에 인수(None)가 전달되어 시그니처 충돌이 발생함.
- **해결**: `server.py` 내 `_handle_tool_call` 호출부에서 `account_id` 존재 유무에 따라 인수가 없는 `self.get_page()` 호출과 인수가 있는 `self.get_page(account_id)` 호출로 분기 처리하여 0-인수 Mock과의 매칭 호환성을 확보함.
- **상태**: 해결
