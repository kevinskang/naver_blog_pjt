# 네이버 블로그 MCP 서버

## 1.1 한줄 요약

> Playwright 웹 자동화로 Claude AI가 네이버 블로그에 직접 포스팅하고 계정/댓글/통계를 관리할 수 있는 MCP 서버

---

## 1.2 프로젝트 목적

네이버 블로그 공식 API가 **2020년 종료**됨에 따라, 웹 브라우저 자동화(Playwright)를 활용하여 Model Context Protocol(MCP) 서버를 구현합니다.

Claude Desktop에서 자연어 명령만으로 네이버 블로그에 글을 작성·수정·삭제하고 카테고리, 댓글, 방문 통계, 멀티 계정 전환을 통합 관리할 수 있도록 합니다.

---

## 1.3 핵심기능

| 기능 | Tool 이름 | 상태 |
|------|-----------|------|
| 블로그 글 작성 및 발행 | `naver_blog_create_post` | ✅ 활성 |
| 카테고리 목록 조회 | `naver_blog_list_categories` | ✅ 활성 |
| 이미지 업로드 (단일/다중) | `naver_blog_create_post` → `images` 파라미터 | ✅ 활성 |
| 글 수정 | `naver_blog_edit_post` | ✅ 활성 |
| 글 삭제 | `naver_blog_delete_post` | ✅ 활성 |
| 글 목록 조회 | `naver_blog_list_posts` | ✅ 활성 |
| 임시저장 목록 및 예약 발행 | `naver_blog_list_drafts`, `naver_blog_publish_draft` | ✅ 활성 |
| 댓글 목록 조회 및 삭제 | `naver_blog_list_comments`, `naver_blog_delete_comment` | ✅ 활성 |
| 블로그 방문 통계 조회 | `naver_blog_get_stats` | ✅ 활성 |
| 멀티 계정 전환 지원 | 각 Tool의 `account_id` 파라미터 | ✅ 활성 |
| 세션 및 로그인 유효성 검사 | `naver_blog_check_session` | ✅ 활성 |
| 세션 자동 관리 | SessionManager | ✅ 활성 |
| 자동 재시도 (지수 백오프) | tenacity | ✅ 활성 |
| 에러 시 자동 스크린샷 | TraceManager / error_handler | ✅ 활성 |

### 글 작성 파라미터

- `title` (필수): 글 제목 (최대 255자)
- `content` (필수): 글 본문
- `category` (선택): 카테고리 이름
- `tags` (선택): 태그 목록 (최대 30개, 각 50자 이내)
- `images` (선택): 이미지 파일 경로 배열 (JPG, PNG, GIF, BMP, HEIC, WebP / 10MB 이하)
- `publish` (선택): 즉시 발행 여부 (기본: true)
- `schedule_time` (선택): 예약 발행 시 예약일시 (형식: "YYYY-MM-DD HH:MM")

## 1.4 기술스택

| 분류 | 기술 | 버전 |
|------|------|------|
| 언어 | Python | 3.13+ |
| 브라우저 자동화 | Playwright | 1.55.0 |
| MCP 프레임워크 | mcp[cli] | 1.20.0+ |
| 데이터 검증 | Pydantic | 2.10.6 |
| 재시도 로직 | Tenacity | 9.1.2 |
| 비동기 파일 I/O | aiofiles | 25.1.0 |
| 패키지 관리 | uv | - |
| 린트 | ruff | 오류 0개 |
| 타입 검사 | pyright | 잔여 11개 (playwright import 및 mcp 데코레이터 구조 제약 이슈) |

### 브라우저 설정
- **User Agent**: Chrome 120 (자동화 감지 우회)
- **Viewport**: 1920×1080
- **Locale**: ko-KR / Asia/Seoul
- **AutomationControlled 비활성화**: `--disable-blink-features=AutomationControlled`
