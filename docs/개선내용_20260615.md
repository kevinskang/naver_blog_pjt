## Phase 1 — 즉시 구현 (1~2일)

### 1-1. 글 삭제 기능 활성화
- **현황**: `tools.py`의 `naver_blog_delete_post` Tool과 핸들러가 주석으로 비활성화된 상태
- **작업 내용**:
  - [x] `src/naver_blog_mcp/mcp/tools.py` 주석 해제
  - [x] `src/naver_blog_mcp/automation/post_actions.py`에 `delete_blog_post()` 함수 구현
  - [x] 삭제 확인 팝업 처리 (iframe 내부 확인 버튼)
  - [x] MCP Tool 핸들러 `handle_delete_post()` 연결
  - [x] 단위 테스트 작성

- **기술 접근**:
  ```
  1. post_url로 글 페이지 이동
  2. iframe#mainFrame 접근
  3. 더보기 메뉴 → 글 삭제 클릭
  4. 확인 팝업에서 확인 버튼 클릭
  5. 삭제 완료 URL 변경 확인
  ```

- **참고**: [[delete-post-research]]

### 1-2. CAPTCHA 대응 전략 개선
- **현황**: 로그인 시 CAPTCHA 발생 → 자동화 불가, 수동 개입 필요
- **작업 내용**:
  - [x] 세션 파일(`auth.json`)이 유효하면 로그인 과정 완전 스킵
  - [x] 세션 만료 시 사용자에게 명확한 안내 메시지 반환
  - [x] `playwright-stealth` 패키지 적용 검토 (→ [[VibeCoding/프로젝트/Naver_blog_pjt/설계/의사결정]] 항목 #1)
  - [x] HEADLESS=false 모드에서 CAPTCHA 수동 처리 후 세션 저장하는 가이드 작성

- **기술 접근**:
  ```
  1. auth.json 파일 존재 및 유효성 검사 강화
  2. 세션 만료 감지 → 재로그인 전 사용자 안내
  3. 브라우저 fingerprint 다양화 (User-Agent rotation)
  ```
