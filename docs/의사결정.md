### DEC-001 | CAPTCHA 자동 처리 전략
- **날짜**: 2026-06-15
- **상태**: 🟡 미결
- **배경**: 현재 네이버 로그인 시 CAPTCHA가 발생하면 자동화가 중단됨. 장기적으로 완전 무인 자동화를 위해 처리 전략이 필요함.
- **선택지**:
  - **A안**: `playwright-stealth` 패키지 적용 — 브라우저 자동화 감지를 최소화하여 CAPTCHA 발생 빈도 감소
  - **B안**: 2captcha / Anti-Captcha 외부 서비스 연동 — 유료 API로 CAPTCHA 자동 풀이
  - **C안**: 현행 유지 — 세션이 유효한 동안은 CAPTCHA 없이 동작, 만료 시 수동 처리 안내
- **고려 사항**:
  - A안: 무료, 효과 불확실, 네이버 정책 변경에 취약
  - B안: 유료(월 수천 원 수준), 높은 성공률, 서비스 의존성 생김
  - C안: 추가 비용 없음, 완전 자동화 불가
- **결정**: 미결 — 실제 운영 환경에서 CAPTCHA 빈도 파악 후 결정 필요
- **영향 범위**: `automation/login.py`, `services/session_manager.py`

## DEC-002: 멀티 계정 지원 구현 방식 (2026-06-15 결정)

**결정**: A안 — 모든 Tool에 `account_id` 파라미터 추가

**배경**: 여러 네이버 계정을 하나의 MCP 서버에서 관리해야 하는 요구사항

**검토 대안**:
- A안: 모든 Tool에 `account_id` 파라미터 추가 ← **채택**
- B안: 계정별 별도 MCP 서버 인스턴스 실행
- C안: `set_active_account` Tool로 전역 계정 전환

**채택 이유**:
- 하나의 서버에서 여러 계정 동시 사용 가능 (B안은 불가)
- Tool 호출 시 명시적으로 계정 지정 → 실수 없음 (C안은 전역 상태 오염 위험)
- `build_tool_metadata`에서 자동 주입 → 기존 Tool 코드 변경 최소화

**구현 세부사항**:
- `_ACCOUNT_ID_PROP` 상수 정의 → `build_tool_metadata`에서 모든 Tool 스키마에 자동 주입
- BrowserContext 풀: `_extra_contexts: Dict[str, Optional[BrowserContext]]`
- 세션 파일: `playwright-state/auth_{account_id}.json`
- 환경변수: `NAVER_ACCOUNT_{ID}_ID`, `NAVER_ACCOUNT_{ID}_PASSWORD`
- 기본 계정(`account_id=""`)은 기존 동작 그대로 유지 (하위 호환)
