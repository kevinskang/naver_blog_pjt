## 조사 배경

### 현재 Tool 정의 상태

`src/naver_blog_mcp/mcp/tools.py`에 이미 정의되어 있음:

```python
"naver_blog_list_categories": {
    "name": "naver_blog_list_categories",
    "description": "네이버 블로그의 카테고리 목록을 가져옵니다.",
    "inputSchema": {
        "type": "object",
        "properties": {},
        "required": [],
    },
}
```

### 구현 상태
- ❌ **미구현**: `handle_list_categories()` 함수가 빈 껍데기만 존재
- ⚠️ 현재는 "아직 구현되지 않았습니다" 메시지만 반환

## 네이버 블로그 카테고리 시스템

### 카테고리 구조

네이버 블로그는 **2단계 계층 구조**를 지원합니다:

```
블로그 루트
├── 카테고리 1 (1차)
│   ├── 서브카테고리 1-1 (2차)
│   └── 서브카테고리 1-2 (2차)
└── 카테고리 2 (1차)
    └── 서브카테고리 2-1 (2차)
```

---

## 카테고리 조회 방법

### 방법 1: 블로그 사이드바에서 조회 (권장)

- 블로그 메인 페이지 접속 → DOM에서 카테고리 링크/텍스트 파싱
- 공개 페이지에서 접근 가능, 카테고리 계층 구조 확인 가능

### 방법 2: 블로그 관리 페이지에서 조회

- 더 구조화된 데이터, 카테고리 ID 명확하게 확인 가능
- 추가 페이지 이동 필요
